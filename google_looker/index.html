🔍 Google Looker

Membuat dashboard BI konsisten dengan semantic model.

Kelas XI

📖 Section 1: Penjelasan mendalam

Konsep Inti

  • Looker memakai semantic layer (LookML) untuk metrik konsisten.
  • Explore memudahkan analisis mandiri user bisnis.
  • Dashboard dapat dijadwalkan dan dibagikan otomatis.
  • Governance data mencegah definisi KPI ganda.

Ringkasan Komponen

KomponenPenjelasanTips Pemula
LookMLModel dimensi dan measure.Gunakan naming konsisten.
ExploreAntarmuka eksplorasi data.Batasi field penting saja.
DashboardVisual KPI untuk stakeholder.Susun dari KPI utama dahulu.

Looker Studio dasar

Pengantar Looker Studio untuk membuat report/dashboard dari data.

Konektor umum: Google Sheet, BigQuery, dan CSV.

🌍 Section 2: Analogi & Kasus Dunia Kerja

Analogi Sehari-hari

  • Looker seperti katalog perpustakaan dengan definisi buku seragam.
  • Explore seperti mesin pencari internal perusahaan.

Kasus Dunia Kerja

Data team SaaS membangun model churn pelanggan di Looker agar semua divisi memakai definisi sama.

Komponen dashboard

Komponen wajib: chart, scorecard, table, filter control, serta fitur sharing/publish untuk kolaborasi.

🛠️ Section 3: Contoh step-by-step

Langkah 1: Definisi view

Buat dimensi dan measure dasar.

view: transaksi {
  dimension: id { primary_key: yes type: number sql: ${TABLE}.id ;; }
  measure: total_omzet { type: sum sql: ${TABLE}.total ;; }
}

Langkah 2: Explore

Expose model untuk dianalisis user.

explore: transaksi {
  label: "Analisis Transaksi"
}

Langkah 3: Dashboard

Dari explore ke visual KPI.

1. Pilih dimensi tanggal dan measure total_omzet
2. Buat line chart
3. Simpan ke dashboard manajemen

Project absensi

Buat dashboard absensi (absensi dashboard project) berisi tingkat kehadiran bulanan, siswa sering absen, dan filter kelas/mapel.
