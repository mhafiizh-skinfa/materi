🟢 Node.js

Menjalankan JavaScript di server untuk API modern dan realtime.

Kelas XI

📖 Section 1: Penjelasan mendalam

Konsep Inti

  • Event loop non-blocking untuk I/O tinggi.
  • Express framework untuk REST API cepat.
  • Middleware untuk validasi, autentikasi, logging.
  • Pengelolaan package dan script menggunakan npm.

Ringkasan Komponen

KomponenPenjelasanTips Pemula
Event LoopMekanisme async utama Node.js.Hindari operasi blocking.
Route APIEndpoint HTTP aplikasi.Gunakan struktur /api/v1.
MiddlewarePemeriksaan sebelum handler.Susun urutan dengan benar.

Node.js wajib

Node.js menjalankan JavaScript di server dengan konsep event loop dan non-blocking I/O.

Manajemen package melalui npm dan package.json. Sistem modul: require (CommonJS) vs import (ES Module).

🌍 Section 2: Analogi & Kasus Dunia Kerja

Analogi Sehari-hari

  • Node.js seperti kasir cepat dengan antrean efisien.
  • Middleware seperti satpam yang memeriksa akses masuk.

Kasus Dunia Kerja

Aplikasi delivery menggunakan Node.js untuk API tracking pesanan real-time.

Stack backend

Gunakan Express untuk server, routing, middleware, koneksi database mysql2, dan pembuatan REST API.

🛠️ Section 3: Contoh step-by-step

Langkah 1: Server dasar

Jalankan server Express.

// Server Express sederhana
const express = require("express");
const app = express();
app.get("/", (req, res) => res.send("Halo Node"));
app.listen(3000);

Langkah 2: Endpoint API

Buat endpoint daftar produk.

app.get("/api/produk", (req, res) => {
  res.json([{ nama: "Roti" }]);
});

Langkah 3: Middleware auth

Cek token sebelum akses route.

// Memvalidasi token request
app.use((req, res, next) => {
  if (!req.headers["x-token"]) return res.status(401).json({ pesan: "Token wajib" });
  next();
});

Contoh REST API

app.get('/siswa', ...);
app.post('/siswa', ...);
app.put('/siswa/:id', ...);
app.delete('/siswa/:id', ...);
// middleware auth/logging dipasang sebelum handler
