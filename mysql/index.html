🗄️ MySQL

Mengelola data relasional aplikasi menggunakan SQL dasar hingga join.

Kelas X

📖 Section 1: Penjelasan mendalam

Konsep Inti

  • Relasi tabel, primary key, foreign key, dan normalisasi dasar.
  • Operasi CRUD dengan SQL.
  • JOIN untuk menggabungkan data antartabel.
  • Index dasar untuk optimasi query baca.

Ringkasan Komponen

KomponenPenjelasanTips Pemula
CREATE TABLEMendefinisikan struktur tabel.Pilih tipe data realistis.
PRIMARY KEYIdentitas unik baris data.Gunakan auto increment.
JOINMenggabungkan data terkait.Pastikan kondisi ON tepat.

Fondasi MySQL

MySQL adalah RDBMS (Relational Database Management System).

Tipe data umum: INT, VARCHAR, TEXT, DATE, BOOLEAN.

DDL: CREATE DATABASE, CREATE TABLE, ALTER TABLE, DROP TABLE. DML: INSERT, SELECT, UPDATE, DELETE.

🌍 Section 2: Analogi & Kasus Dunia Kerja

Analogi Sehari-hari

  • MySQL seperti lemari arsip dengan laci saling terkait.
  • Seperti buku induk sekolah untuk data siswa dan nilai.

Kasus Dunia Kerja

Aplikasi kasir minimarket menyimpan produk, transaksi, dan stok dengan MySQL.

Query penting

Filter dan analisis dasar: WHERE, ORDER BY, GROUP BY, HAVING, LIMIT. Relasi tabel: INNER JOIN, LEFT JOIN, RIGHT JOIN.

🛠️ Section 3: Contoh step-by-step

Langkah 1: Buat tabel

Membuat tabel siswa.

-- Membuat tabel siswa
CREATE TABLE siswa (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nama VARCHAR(100),
  kelas VARCHAR(20)
);

Langkah 2: Isi data

Masukkan data awal.

-- Menambahkan data
INSERT INTO siswa (nama, kelas) VALUES ("Alya","X RPL 1"), ("Bima","X RPL 2");

Langkah 3: Ambil data

Filter data sesuai kebutuhan.

-- Mengambil data siswa kelas tertentu
SELECT nama FROM siswa WHERE kelas = "X RPL 1";

Contoh database sekolah

tabel: siswa, guru, mapel, nilai, absensi
SELECT s.nama, m.nama_mapel, n.nilai
FROM nilai n
INNER JOIN siswa s ON s.id_siswa = n.id_siswa
INNER JOIN mapel m ON m.id_mapel = n.id_mapel
WHERE n.nilai >= 75
ORDER BY n.nilai DESC
LIMIT 10;
