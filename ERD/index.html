🧩 ERD

Merancang struktur database dari entitas, atribut, dan relasi.

Kelas XI

📖 Section 1: Penjelasan mendalam

Konsep Inti

  • Komponen ERD: entitas, atribut, relasi, kardinalitas.
  • Penentuan primary key dan foreign key.
  • Relasi 1:1, 1:N, N:M dan tabel penghubung.
  • Normalisasi awal sebelum implementasi database.

Ringkasan Komponen

KomponenPenjelasanTips Pemula
EntitasObjek utama data (contoh: Siswa).Nama pakai kata benda.
AtributDetail dari entitas.Pisahkan wajib vs opsional.
RelasiHubungan antar entitas.Tentukan kardinalitas jelas.

Komponen ERD wajib

Komponen utama: entitas, atribut, dan relasi.

Jenis atribut: simple, composite, derived, multivalued.

Kardinalitas: one-to-one, one-to-many, many-to-many.

🌍 Section 2: Analogi & Kasus Dunia Kerja

Analogi Sehari-hari

  • ERD seperti blueprint gedung sebelum pembangunan.
  • ERD seperti bagan keluarga yang menunjukkan relasi anggota.

Kasus Dunia Kerja

Database engineer marketplace merancang ERD pelanggan, produk, dan pesanan sebelum coding backend.

Notasi ERD

Gunakan notasi Crow's Foot (praktis desain database) dan Chen (konseptual akademik) agar siswa paham dua pendekatan populer.

🛠️ Section 3: Contoh step-by-step

Langkah 1: Identifikasi entitas

Ambil studi kasus perpustakaan.

Entitas: Anggota, Buku, Peminjaman

Langkah 2: Tentukan relasi

Hubungkan entitas sesuai kardinalitas.

Anggota 1 --- n Peminjaman
Buku 1 --- n Peminjaman

Langkah 3: Turunkan ke tabel

Konversi ERD menjadi skema SQL.

anggota(id_anggota PK, nama)
buku(id_buku PK, judul)
peminjaman(id_pinjam PK, id_anggota FK, id_buku FK)

Contoh perpustakaan + mapping fisik

Entitas: Buku, Anggota, Peminjaman. Relasi Buku-Many to Many-Anggota diselesaikan tabel penghubung Peminjaman. Tahap ini disebut konversi ke Physical Table di SQL.

CREATE TABLE anggota (...);
CREATE TABLE buku (...);
CREATE TABLE peminjaman (
  id SERIAL PRIMARY KEY,
  anggota_id INT REFERENCES anggota(id),
  buku_id INT REFERENCES buku(id),
  tgl_pinjam DATE
);
