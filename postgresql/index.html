🐘 PostgreSQL

Database relasional kuat untuk transaksi aman dan analisis data.

Kelas X

📖 Section 1: Penjelasan mendalam

Konsep Inti

  • Transaksi ACID untuk konsistensi data.
  • Fitur lanjutan: JSONB, CTE, window function.
  • Index dan query plan untuk performa.
  • Kapan memilih PostgreSQL untuk sistem skala besar.

Ringkasan Komponen

KomponenPenjelasanTips Pemula
BEGIN/COMMITMembungkus operasi atomik.Gunakan rollback saat error.
JSONBMenyimpan data semi-terstruktur.Index jika sering difilter.
CTEMemecah query kompleks.Beri nama CTE deskriptif.

Perbandingan PostgreSQL vs MySQL

AspekPostgreSQLMySQL
Fitur lanjutSangat kuat (JSONB, CTE, window function)Baik untuk kebutuhan umum

Tipe penting: SERIAL, JSONB, ARRAY, UUID.

🌍 Section 2: Analogi & Kasus Dunia Kerja

Analogi Sehari-hari

  • PostgreSQL seperti brankas bank yang ketat menjaga konsistensi.
  • Seperti perpustakaan digital dengan katalog sangat detail.

Kasus Dunia Kerja

Perusahaan fintech menggunakan PostgreSQL untuk pencatatan transaksi yang wajib akurat.

Setup dasar

Instal PostgreSQL, lalu gunakan terminal psql: \l (list DB), \c nama_db (connect), \dt (list tabel), \q (keluar).

🛠️ Section 3: Contoh step-by-step

Langkah 1: Buat tabel

Siapkan tabel transaksi.

-- Membuat tabel transaksi
CREATE TABLE transaksi (
  id SERIAL PRIMARY KEY,
  total NUMERIC(12,2),
  dibuat_pada TIMESTAMP DEFAULT NOW()
);

Langkah 2: Transaksi aman

Update dalam blok transaksi.

-- Menjalankan transaksi aman
BEGIN;
UPDATE saldo SET jumlah = jumlah - 50000 WHERE id_user = 1;
COMMIT;

Langkah 3: Ringkasan data

Hitung total per bulan.

-- Ringkasan bulanan
SELECT DATE_TRUNC('month', dibuat_pada) bulan, SUM(total)
FROM transaksi GROUP BY bulan;

Contoh online shop

CREATE TABLE products (
  id UUID PRIMARY KEY,
  name VARCHAR(100),
  price INT,
  tags TEXT[],
  spec JSONB
);
CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  product_id UUID REFERENCES products(id),
  qty INT
);
