🗺️ DFD

Memetakan arus data sistem agar proses bisnis mudah dianalisis.

Kelas XI

📖 Section 1: Penjelasan mendalam

Konsep Inti

  • Komponen DFD: external entity, process, data store, data flow.
  • Level DFD: context diagram, level 0, level 1.
  • DFD fokus arus data, bukan urutan kontrol seperti flowchart.
  • Penamaan proses dan data flow harus konsisten.

Ringkasan Komponen

KomponenPenjelasanTips Pemula
External EntityAktor di luar sistem.Contoh: Siswa/Admin.
ProcessTransformasi data.Gunakan format kata kerja.
Data StorePenyimpanan data.Selaraskan dengan DB nyata.

Elemen DFD wajib

  • Entitas eksternal: pihak luar sistem.
  • Proses: transformasi data.
  • Data store: penyimpanan data.
  • Arus data: aliran informasi antar elemen.

🌍 Section 2: Analogi & Kasus Dunia Kerja

Analogi Sehari-hari

  • DFD seperti jalur paket dari pengirim ke penerima.
  • DFD seperti alur surat masuk dari TU ke kepala sekolah.

Kasus Dunia Kerja

System analyst menyusun DFD PPDB online agar tim memahami alur data pendaftaran dan verifikasi.

Level DFD

Context Diagram (Level 0) menampilkan sistem sebagai satu proses besar. Level 1 memecah proses utama. Level 2 merinci subproses kritis.

🛠️ Section 3: Contoh step-by-step

Langkah 1: Context diagram

Tampilkan sistem sebagai 1 proses utama.

[Siswa] --Data--> (Sistem PPDB) --Status--> [Siswa]
[Admin] --Verifikasi--> (Sistem PPDB)

Langkah 2: DFD level 0

Pecah proses utama jadi subproses.

P1 Terima Form
P2 Verifikasi Berkas
P3 Umumkan Hasil
Data Store: D1 Data Pendaftar

Langkah 3: Pseudocode proses

Uji logika verifikasi.

Mulai
 input data
 jika berkas lengkap maka status <- "Lolos"
 selain itu status <- "Perbaiki"
 simpan status
Selesai

Kesalahan umum

  • Arus data tanpa nama.
  • Entitas eksternal dihubungkan langsung ke data store tanpa proses.
  • Level 1/2 tidak seimbang dengan context diagram.
