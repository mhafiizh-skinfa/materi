🔀 Flowchart

Memvisualisasikan alur solusi agar mudah dipahami sebelum diubah menjadi kode.

Kelas X

📖 Section 1: Penjelasan mendalam

Konsep Inti

  • Simbol utama flowchart: terminator, process, decision, input/output.
  • Aturan alur: satu arah, minim garis silang, label jelas.
  • Flowchart membantu validasi logika sebelum coding.
  • Flowchart bisa diturunkan menjadi pseudocode terstruktur.

Ringkasan Komponen

KomponenPenjelasanTips Pemula
TerminatorSimbol mulai/selesai.Wajib ada di awal dan akhir.
DecisionTitik percabangan kondisi.Pastikan cabang Ya/Tidak jelas.
ProcessLangkah proses utama.Gunakan kata kerja aktif.

Tabel simbol lengkap

SimbolFungsi
TerminalMulai/Selesai proses.
ProsesLangkah pengolahan data.
KeputusanPercabangan Ya/Tidak.
Input/OutputMenerima atau menampilkan data.
ConnectorMenyambungkan alur antarbagian/halaman.

Aturan: alur atas-ke-bawah/kiri-ke-kanan, satu simbol satu fungsi, panah harus jelas, dan keputusan minimal dua cabang.

🌍 Section 2: Analogi & Kasus Dunia Kerja

Analogi Sehari-hari

  • Seperti peta perjalanan dengan titik belok keputusan.
  • Seperti SOP piket kelas agar urutan kerja tidak tertukar.

Kasus Dunia Kerja

Analis sistem membuat flowchart proses klaim agar user, manajer, dan programmer punya pemahaman alur yang sama.

Jenis flowchart

Flowchart sistem menggambarkan alur antarunit (user, aplikasi, database). Flowchart program fokus ke logika detail kode pada satu modul.

Kasus nyata: flowchart login, cek kelulusan, dan proses ATM (cek saldo/tarik tunai).

🛠️ Section 3: Contoh step-by-step

Langkah 1: Diagram teks

Bangun flow cek kelulusan.

[Mulai] -> [Input nilai] -> {nilai >= 75?}
 Ya -> [Lulus] -> [Selesai]
 Tidak -> [Remedial] -> [Selesai]

Langkah 2: Tambah aturan

Masukkan syarat absensi.

[Input nilai, hadir] -> {nilai>=75?}
 Ya -> {hadir>=80?} -> [Lulus/Tidak]
 Tidak -> [Remedial]

Langkah 3: Jadi pseudocode

Turunkan flow ke langkah logis.

Mulai
 input nilai, hadir
 jika nilai>=75 dan hadir>=80 maka cetak "Lulus"
 selain itu cetak "Remedial/Tidak Lulus"
Selesai

Contoh alur singkat

Login: Start -> input user/pass -> valid? -> masuk dashboard / tampilkan error -> End
Cek kelulusan: input nilai akhir -> nilai >= KKM? -> Lulus/Tidak Lulus
ATM: masukkan kartu+PIN -> valid? -> pilih transaksi -> cetak struk -> End
