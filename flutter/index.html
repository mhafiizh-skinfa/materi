📱 Flutter (Mobile Developer)

Membuat aplikasi mobile lintas platform memakai widget dan Dart.

Kelas XI & XII

📖 Section 1: Penjelasan mendalam

Konsep Inti

  • Struktur project Flutter, widget tree, dan lifecycle dasar.
  • Perbedaan StatelessWidget vs StatefulWidget.
  • Navigasi antar halaman dan manajemen state awal.
  • Best practice struktur folder untuk fitur mobile.

Ringkasan Komponen

KomponenPenjelasanTips Pemula
WidgetKomponen UI Flutter.Pecah menjadi widget kecil reusable.
StateData yang berubah di UI.Mulai dari setState lalu naik level.
NavigationPerpindahan halaman.Gunakan route bernama untuk app besar.

Pemetaan XI dan XII

Kelas XI fokus fondasi widget, state dasar, dan navigasi. Kelas XII wajib mencakup Provider/Riverpod, request HTTP memakai dio/http, integrasi REST API, penyimpanan lokal SQLite (sqflite), dan dasar push notification.

🌍 Section 2: Analogi & Kasus Dunia Kerja

Analogi Sehari-hari

  • Flutter seperti lego digital: widget kecil jadi aplikasi utuh.
  • State seperti papan skor yang berubah seiring interaksi user.

Kasus Dunia Kerja

Perusahaan retail membuat satu aplikasi Android/iOS dari satu codebase Flutter.

Kasus dunia kerja

Studi kasus: aplikasi absensi sekolah yang terhubung backend API untuk login, kirim kehadiran, dan sinkronisasi data.

🛠️ Section 3: Contoh step-by-step

Langkah 1: App dasar

Menampilkan teks sederhana.

import "package:flutter/material.dart";

void main() => runApp(const MaterialApp(home: Scaffold(body: Center(child: Text("Halo Flutter")))));

Langkah 2: Stateful counter

Update UI dengan setState.

// Counter sederhana
int hitung = 0;
// Panggil setState(() => hitung++) saat tombol ditekan

Langkah 3: Navigasi

Pindah dari halaman utama ke detail.

Navigator.push(context, MaterialPageRoute(builder: (_) => const DetailPage()));

Contoh integrasi

// state management: Provider/Riverpod
// HTTP request: dio.get(...) atau http.get(...)
// simpan offline: sqflite
// push notification: Firebase Cloud Messaging basic setup
// sinkron ke REST API absensi sekolah

🚀 Materi Lanjutan (Kelas XII)

  • State management lanjutan: Provider, Riverpod, BLoC untuk aplikasi skala produksi.
  • Clean Architecture + repository pattern agar kode modular dan mudah diuji.
  • Integrasi API kompleks, cache offline (Hive/SQLite), dan sinkronisasi data.
  • Testing lengkap: unit test, widget test, integration test, dan CI/CD pipeline.
  • Rilis production: signing Android/iOS, optimasi performa, dan crash monitoring (Firebase).
